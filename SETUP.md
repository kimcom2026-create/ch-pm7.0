# راهنمای راه‌اندازی همگام‌سازی (گوگل درایو) و پشتیبان‌گیری محلی

این برنامه یک فایل HTML واحد است (`V7.0.html`). همهٔ داده‌ها در مرورگر (IndexedDB) ذخیره می‌شود.
دو قابلیت جدید اضافه شده است:

1. **همگام‌سازی با گوگل درایو** — تغییرات به‌صورت خودکار در یک پوشهٔ مخفیِ مخصوص برنامه روی
   گوگل درایو شما ذخیره می‌شود و هنگام باز کردن برنامه، داده‌ها از درایو خوانده می‌شوند. در نتیجه
   روی کامپیوتر و موبایل داده‌های یکسان دارید.
2. **پشتیبان‌گیری در پوشهٔ محلی** — یک پوشه روی هارد انتخاب می‌کنید و با هر تغییر، یک فایل
   پشتیبان `project-manager-backup.json` به‌صورت خودکار در آن نوشته می‌شود (فقط Chrome/Edge دسکتاپ).

> هر دو قابلیت نیاز دارند برنامه از یک **آدرس وب** باز شود (نه با دابل‌کلیک روی فایل).
> گوگل اجازهٔ ورود (OAuth) را برای آدرس‌های `file://` نمی‌دهد.

---

## بخش ۱ — انتشار رایگان روی GitHub Pages

> هزینه‌ای ندارد و به سرور نیاز نیست؛ فقط همین یک فایل HTML سرو می‌شود.
> توجه: GitHub Pages رایگان عمومی است (هرکسی می‌تواند صفحه را باز کند) اما **داده‌های شما
> خصوصی می‌ماند**، چون داده در گوگل درایو شخصی شما و پشت ورود گوگل شماست.

1. در <https://github.com> یک حساب بسازید/وارد شوید.
2. **New repository** → یک نام دلخواه (مثلاً `pm`) → **Public** → Create.
3. فایل `V7.0.html` را به **`index.html`** تغییر نام دهید و در ریپازیتوری آپلود کنید
   (Add file → Upload files → Commit).
4. **Settings → Pages** → در بخش *Branch* گزینهٔ `main` و پوشهٔ `/ (root)` را انتخاب و **Save**.
5. بعد از چند دقیقه آدرس صفحه نمایش داده می‌شود، مثل:
   `https://USERNAME.github.io/pm/`
   این آدرس را یادداشت کنید (در بخش بعد لازم است). همین آدرس را روی موبایل هم باز می‌کنید.

## بخش ۲ — ساخت دسترسی گوگل درایو (Google Cloud Console)

این کار یک‌بار انجام می‌شود.

1. به <https://console.cloud.google.com> بروید.
2. بالای صفحه **Select a project → New Project** → یک نام بدهید → Create.
3. منوی جستجو → **Google Drive API** → **Enable**.
4. منوی سمت چپ → **APIs & Services → OAuth consent screen**:
   - نوع: **External** → Create.
   - نام برنامه، ایمیل پشتیبانی و ایمیل توسعه‌دهنده را وارد کنید → Save and Continue.
   - در مرحلهٔ **Test users** روی *Add users* بزنید و **ایمیل گوگل خودتان** را اضافه کنید
     (همان حسابی که داده در درایو آن ذخیره می‌شود) → Save.
5. منوی سمت چپ → **APIs & Services → Credentials**:
   - **Create Credentials → API key**. کلید ساخته‌شده را کپی کنید (بعداً در برنامه وارد می‌شود).
     (اختیاری: روی کلید، Restrict key → API restrictions → فقط Google Drive API.)
   - دوباره **Create Credentials → OAuth client ID**:
     - Application type: **Web application**.
     - در **Authorized JavaScript origins** دقیقاً مبدأ آدرس Pages را اضافه کنید
       (بدون مسیر انتهایی)، مثلاً:
       `https://USERNAME.github.io`
       و برای تست محلی هم می‌توانید اضافه کنید: `http://localhost:8000`
     - Create. مقدار **Client ID** را کپی کنید.

## بخش ۳ — اتصال در خود برنامه

1. آدرس Pages را در مرورگر باز کنید (یا برای تست، `http://localhost:8000/V7.0.html`).
2. در صفحهٔ نخست، روی **آیکن چرخ‌دندهٔ بالا-چپ** بزنید.
3. **Client ID** و **API Key** را در کادرهای مربوطه وارد کنید.
4. تیک «همگام‌سازی خودکار با گوگل درایو فعال باشد» را بزنید → **ذخیره تنظیمات**.
5. روی **اتصال به گوگل درایو** بزنید و با حساب گوگلِ تست‌شده وارد شوید و دسترسی را تأیید کنید.
6. از این پس تغییرات به‌صورت خودکار همگام می‌شوند. روی موبایل هم همان آدرس را باز کنید،
   همان مراحل را طی کنید؛ داده‌ها یکی می‌شوند.

> اگر هنگام ورود پیام «این برنامه تأیید نشده» دیدید، چون برنامه در حالت تست است و شما
> Test user هستید طبیعی است؛ روی *Advanced → Go to ...* بزنید و ادامه دهید.

---

## بخش ۴ — پشتیبان‌گیری در پوشهٔ محلی (دسکتاپ Chrome/Edge)

1. همان پنجرهٔ تنظیمات (آیکن چرخ‌دنده) → بخش «پشتیبان‌گیری در پوشهٔ محلی».
2. **انتخاب پوشه** → پوشهٔ دلخواه روی هارد را انتخاب و دسترسی را تأیید کنید.
3. از این پس با هر تغییر، فایل `project-manager-backup.json` به‌صورت خودکار در آن پوشه
   به‌روزرسانی می‌شود. با دکمهٔ «پشتیبان‌گیری فوری» هم می‌توانید دستی ذخیره کنید.

> این قابلیت روی موبایل و در مرورگرهای Safari/Firefox در دسترس نیست و به‌صورت خودکار پنهان می‌شود.

---

## نکات

- **محرمانگی کلیدها:** Client ID و API Key در برنامه‌های سمت‌کلاینت محرمانه نیستند؛ امنیت از طریق
  ورود گوگل، فهرست مبداهای مجاز و فهرست Test users تأمین می‌شود. بنابراین قرار دادن فایل در یک
  ریپازیتوری عمومی مشکلی ایجاد نمی‌کند.
- **حل تعارض:** هنگام باز شدن برنامه، اگر نسخهٔ درایو جدیدتر باشد جایگزین می‌شود (با تأیید شما)؛
  در غیر این صورت نسخهٔ این دستگاه به درایو ارسال می‌شود. برای استفادهٔ یک‌کاربره روی دو دستگاه کافی است.
- **بدون اینترنت:** اگر متصل نباشید، داده همچنان در مرورگر (IndexedDB) و در پوشهٔ محلی ذخیره می‌ماند و
  در اتصال بعدی همگام می‌شود.
